# Mars Rover Docking System: Technical Specification

## System Overview
This embedded system facilitates autonomous docking for a Mars Rover prototype. It utilizes a load cell for weight-based triggering, dual-axis stepper motor control for positioning, and a relay-controlled charging sequence. The logic is built for Arduino-compatible controllers using non-blocking timing.

---

## Block Diagram

The system architecture connects a high-voltage AC source through an SMPS to power both the logic (via buck converter) and the high-draw components (motors and chargers).

---

## Hardware Components
* **Microcontroller:** Arduino Uno (or compatible board)
* **Motion Control:** CNC Shield, 2× Stepper Motors, 2× Limit Switches
* **Sensors:** HX711 Amplifier + Load Cell (Weight Sensing)
* **Isolation/Control:** Single Channel Relay, 12V Optocoupler
* **Power Management:** 220V AC to 12V DC SMPS, DC-DC Buck Converter (5V), iMAX Balance Charger

## Control Logic and State Machine

### 1. Motion Control (Weight Trigger)
* **Condition: Weight > 200**
    * Initiate **Forward** motion.
    * In this state, limit switches are ignored to allow full engagement.
* **Condition: Weight < 200**
    * Initiate **Reverse** motion (Homing).
    * Each axis stops independently once its respective limit switch is triggered.
    * A system-wide reset occurs only when **both** limit switches are engaged.

### 2. Emergency & Sequence Stop
* **Condition: STOP Signal (Pin 12) is HIGH**
    * Immediate cessation of all motor movement.
    * Execution of a timed relay sequence (non-blocking via `millis()`).
    * The relay sequence runs exactly once per stop trigger.

---

## Technical Implementation Notes

* **Non-Blocking Timing:** The system uses `millis()` rather than `delay()` for the relay sequence to ensure the microcontroller remains responsive to sensor inputs.
* **Constant Velocity:** Motors operate at a fixed step rate; no acceleration/deceleration ramps are implemented in the current profile.
* **Signal Isolation:** An optocoupler is utilized on the 12V line to protect the Arduino logic from voltage spikes and noise generated by the battery/charging circuit.
* **Power Rail:** * **12V Rail:** Powers the CNC Shield (Motors), iMAX Charger, and LED Strip.
    * **5V Rail:** Regulated via Buck Converter to power the Arduino and HX711 module.
